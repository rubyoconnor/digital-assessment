Pasta Planning

Brief

To create a program in which someone is able to enter customer details, kinds of pasta ordered and the pick-up or delivery details into a computer and have it display the delivery details, the itemised order, and total cost.

Initial Project Backlog

Describing Relevant Implications

Usability

Usability means that the user can intutively learn the program quickly. Options are easy to understand and it is easy to find appropriate actions. The program has error prevention methods and communicates to the user about then.

Functionality

Functionality means that the program must work properly. This means it does not crash, it does not do incorrect arithemetic and variables are all reset once order is complete. The program should work as intended.

Sustainability and Future Proofing

Sustainability and future proofing is making the program so that it is able to be updated and maintained. The program should be able to have new feature added reasonably easily. It should be able to be handed over to another developer and they should be able to understand it quickly.

Addressing Relevant Implications

Usability

In my program, the Usability implication can be seen as I created validation functions to ensure appropriate entry. All the user inputs are validated with clear messages to help the user fix their error and move foward with the appropriate actions. I also used a simple menu system which was like a home screen. This is make it easy for the users to work the program as the home screen makes it clear to the user the options availible.

Functionality

In my program, I have shown the functionality implication as I thoroughly tested at the end of each sprint to make sure that everything was working as I intended. I also ran a full user test with a real user to test the program under unpredictable conditions. This is so that I can make sure that my program does not crash and is working as intended.

Sustainability and Future Proofing

The sustainability and future proofing implication is shown in my program as I have used a function based build where every each feature is a function. For each new feature, you can create a new function and it does not impact what is already there. This makes it easy to add on new features in the future. My whole program is also commented and doctyped which would make it easier if another developer was to try to understand it. features.
